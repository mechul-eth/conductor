# Code of Conduct

Conductor is a community project. We want it to be a place where people can contribute, ask questions, and learn without running into hostile or discouraging behavior.

This code of conduct applies in all project spaces — GitHub issues, pull requests, discussions, and anywhere you're representing the project.

---

## What we expect

**Be direct, not cruel.** Disagreeing with a technical approach is fine. Dismissing someone because of who they are is not.

**Give context when you push back.** If something is wrong, explain why. "This is wrong" teaches nothing. "This breaks the loop-safety guarantee because X" actually helps.

**Assume the other person is trying.** Most people who ask a question or open a PR are doing their best. Start there.

**Keep it relevant.** This is a project about AI routing and orchestration. Conversations should stay on that. Personal conflicts, political debates, and off-topic fights don't belong here.

---

## What we don't allow

- Harassment, abuse, or targeted personal attacks
- Discriminatory comments based on gender, race, sexuality, age, religion, disability, or background
- Publishing someone's private information without their permission
- Sustained disruption of discussions or reviews
- Threats or intimidation, real or implied

---

## If something goes wrong

If you see behavior that violates this code of conduct, or if you're on the receiving end of it, please email the maintainer directly or open a [GitHub Discussion](https://github.com/mechul-eth/conductor/discussions) if you're comfortable doing so publicly.

Reports are kept confidential. We will respond within 72 hours and explain what action was taken (or why no action was taken).

People who repeatedly or severely violate this code of conduct will be removed from the project community.

---

## For contributors

By participating in this project, you agree to follow these guidelines. This includes contributing code, opening issues, commenting on PRs, or participating in any community space.
